## 🗓️ 12-дневный план

### 🔴 Дни 1–2: Go основы + первый HTTP сервер

#### День 1 — Освежаем Go: указатели, maps, error handling
**Теория (30 мин):** указатели `*` и `&`, maps, тип `error`, паттерн `if err != nil`

**Задания:**

```
Задание 1:
Напиши функцию которая принимает имя и возраст,
возвращает строку "Привет, Алибек! Тебе 20 лет"
и error если возраст меньше 0.

Задание 2:
Создай map где ключ — название города, значение — население.
Напиши функцию которая принимает название города
и возвращает население или error если города нет в map.
```

---

#### День 2 — Первый HTTP сервер на чистом Go (без фреймворка)
**Теория (20 мин):** `net/http`, handlers, `json.Marshal` / `json.Unmarshal`

**Задание:**

```
Подними сервер на порту 8080. Сделай 2 endpoint'а:

GET  /hello → возвращает JSON: {"message": "Hello World"}
POST /sum   → принимает JSON: {"a": 5, "b": 3}
             → возвращает JSON: {"result": 8}

Проверяй через Postman или curl.
```

> 💡 Зачем день без Gin? Чтобы понять что Gin делает под капотом — тогда не потеряешься на хакатоне.

---

### 🟠 Дни 3–4: Gin фреймворк

#### День 3 — Gin основы: роутинг, параметры, JSON
**Теория (20 мин):** `gin.Default()`, `c.JSON()`, `c.Param()`, `c.ShouldBindJSON()`

**Задание:**

```
Перепиши вчерашний сервер на Gin.
Добавь третий endpoint:

GET /user/:name → возвращает {"greeting": "Привет, {name}!"}

Добавь middleware который печатает в консоль
время каждого запроса.
```

---

#### День 4 — Gin: структуры, валидация, коды ответов
**Теория (20 мин):** `binding:"required"`, статус коды, группы роутов

**Задание:**

```
Сделай мини API для списка задач (Todo).
Структура Task: ID, Title, Done (bool)

POST /tasks      → создать задачу (Title обязателен!)
GET  /tasks      → вернуть все задачи
GET  /tasks/:id  → вернуть одну задачу
PUT  /tasks/:id  → отметить как выполненную

Храни задачи в slice (без базы пока).
Если задача не найдена → возвращай 404.
```

---

### 🟡 Дни 5–6: PostgreSQL подключение

#### День 5 — Подключаем PostgreSQL через pgx
**Теория (20 мин):** `pgx` драйвер, `db.Query`, `db.Exec`, `.env` файл

**Задание:**

```
Подключи PostgreSQL к серверу.
Создай таблицу users:
  id    SERIAL PRIMARY KEY
  name  TEXT
  email TEXT

Сделай endpoints:
POST /users     → создать пользователя
GET  /users     → получить всех из БД
GET  /users/:id → получить одного по id
```

---

#### День 6 — Нормальная структура проекта
**Теория (20 мин):** передача `db` через контекст Gin, разделение на файлы

**Задание:**

```
Раздели проект на папки:
  /handlers  → логика endpoints
  /db        → работа с базой
  main.go    → только запуск сервера

Перенеси код users в эту структуру.
Добавь endpoint DELETE /users/:id.

⭐ Это структура которую будешь использовать на хакатоне!
```

---

### 🟢 Дни 7–8: AI интеграция — главная роль в команде

#### День 7 — Вызываем Gemini API из Go
**Теория (20 мин):** Gemini API (бесплатный!), HTTP POST из Go, Bearer token

> Сначала зарегистрируйся: [aistudio.google.com](https://aistudio.google.com) → получи API ключ

**Задание:**

```
Напиши функцию sendToGemini(prompt string) string
которая отправляет текст в Gemini и возвращает ответ.

Затем сделай endpoint:
POST /ask
Body:   {"question": "Что такое машинное обучение?"}
Ответ:  {"answer": "...ответ от Gemini..."}

Храни API ключ в .env файле, читай через os.Getenv()
```

---

#### День 8 — Чатбот с историей сообщений
**Теория (15 мин):** как работает контекст в Chat API (массив messages)

**Задание:**

```
Улучши /ask до полноценного чата.

POST /chat
Body: {
  "session_id": "user123",
  "message": "Привет!"
}

Сервер должен помнить историю по session_id.
Каждый новый вопрос отправляется вместе со всей
историей разговора в Gemini.

Проверка: задай вопрос, потом спроси "что я только
что спросил?" — бот должен вспомнить.
```

---

### 🔵 Дни 9–10: Собираем всё вместе

#### День 9 — Backend + AI + БД в одном проекте

**Задание:**

```
Собираем реальный мини-продукт:
AI-ассистент который помнит разговоры в базе данных.

Таблица conversations:
  id, session_id, role (user/assistant), message, created_at

Логика:
1. Пришёл запрос POST /chat с session_id и message
2. Загрузи историю сессии из БД
3. Добавь новое сообщение
4. Отправь всё в Gemini
5. Сохрани ответ в БД
6. Верни ответ пользователю

GET /history/:session_id → вся история чата
```

---

#### День 10 — CORS + командная интеграция
**Теория (15 мин):** CORS в Gin, зачем нужен, как настроить

**Задание:**

```
Добавь CORS middleware чтобы фронтендер из команды
мог делать запросы к твоему серверу.

Вместе с командой: сделай простую страницу
которая отправляет сообщение на /chat и показывает ответ.
Это ваша первая интеграция!
```

## 💡 Главные правила

1. **20 минут теории → остальное практика** каждый день
2. **Набирай код руками** — не копируй, мышечная память спасёт на хакатоне
3. **На хакатоне важно работающее демо**, а не чистый код
4. **Узкая идея > широкая** — лучше простой чатбот который работает, чем сложная система которая не запускается
